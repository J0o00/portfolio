Research & Innovation

Patent Pending

Context-Aware Digital Triplet System for Electrical Motors

Indian Patent App: 202641051670 Status: Under Review Download Technical Summary PDF ↓

Abstract

Conventional digital twins provide simulation fidelity but lack the closed-loop actuation layer needed for real-time motor health intervention. Under variable electrical load, the gap between simulated state and physical state grows, causing delayed fault response and preventable downtime.

Novel Contribution

The system introduces a three-layer "triplet" model that closes the loop between prediction and physical response, entirely eliminating the read-only limitation of conventional twins. By integrating direct actuation pathways, the system preempts thermal runaway.

Applications

Heavy industrial drives, EV powertrain prognostics, and autonomous manufacturing plants.

Design Principles

Engineering Approach

01. System-Level Thinking

Every component exists within a system. I design at the interface boundaries — between power and control, between embedded hardware and cloud supervision — because that is where most failures originate and most improvements are possible.

02. Simulate Before You Fabricate

MATLAB-based modelling and PLCSIM validation precede physical implementation. Catching a fault in simulation costs nothing; finding it on a live system costs everything.

03. Fault Tolerance Is Non-Negotiable

Industrial and EV systems operate in environments where failure has physical consequences. Every control design I produce includes structured fault states, watchdog logic, and defined safe-mode behaviour.

Engineering Philosophy

Designing Under Constraints

Hardware always behaves differently than simulation.

Sensor noise is never zero. Parasitic inductance matters. Simulations prove theoretical feasibility, but physical testing and validation are required to prove viability.

Fault handling must be designed before deployment.

A system's engineering quality is defined by how gracefully it fails, not how perfectly it runs under ideal conditions. Watchdog timers and mechanical interlocks are not optional.

Reliability beats complexity.

An elegant, flat control logic structure that a floor technician can easily debug at 3 AM is vastly superior to a complex, black-box AI model that no one on the ground understands.

Completed

Siemens PLC Automation

GitHub Documentation

Overview

Industrial automation internship at the Siemens Centre of Excellence, Anna University (Jun – Jul 2025). Programmed and validated PLC-based control workflows for industrial process automation scenarios.

Problem

Manual process sequences in industrial lines introduce human error, slow throughput, and unsafe operating conditions. The goal was to design a fully automated, fault-safe control sequence using programmable logic controllers.

Trade-offs & Constraints

  • Interrupts vs. Polling: Chose polling for non-critical sensor checks to reduce interrupt overhead, trading 10ms latency for system stability.
  • Complexity vs. Maintainability: Kept ladder logic flat to ensure floor technicians could troubleshoot it without advanced training.
  • Constraints: Real-time cycle time limits and fail-safe interlocks with legacy HMI systems.

System Architecture

Technologies Used

Siemens S7-1200 PLC TIA Portal Ladder Logic SCADA HMI Panel

Key Engineering Decisions

  • Why PLC instead of a Microcontroller? Required industrial-grade electrical noise immunity and native 24V I/O for direct actuator control.
  • Why TIA Portal? Unified environment allowed simultaneous development of the PLC logic and HMI screens, ensuring data tags were perfectly synchronized.

Implementation

Programmed ladder logic sequences for sequential actuator control and fault-safe operation. Designed SCADA and HMI interfaces for live process monitoring and operator visualization. Structured the control architecture for reliability under variable industrial load conditions.

Engineering Failure Log

Failure: Sensor signal loss during high-load switching.

Investigation: TIA Portal trace revealed inductive kickback causing false lows on the PLC input module.

Solution: Implemented structured error-state logic branches and a 500ms watchdog timer.

Result: Safe system halt with HMI alerts instead of indefinite sequence freezing.

System Impact

  • Process Consistency: Automated repetitive control sequences, reducing operator dependency and improving overall process consistency.
  • Safety: Added fault-tolerant logic and mechanical interlocks to prevent hazardous collisions during sensor failure.

Completed

eBAJA EV Powertrain

GitHub Documentation

Overview

Contributed to the Battery Management System (BMS) design and vehicle electronics integration for the eBAJA EV Competition 2025 — an off-road electric vehicle challenge.

Problem

Off-road EVs demand robust battery monitoring and thermal management under extreme vibration, variable load, and high peak-current conditions. A failure in the BMS can lead to thermal runaway or complete vehicle shutdown mid-race.

Trade-offs & Constraints

  • Accuracy vs. Processing Speed: Settled on a 10Hz sampling rate to prevent the main control loop from overrunning its time budget.
  • Battery Life vs. Telemetry: Reduced active monitoring during idle states to preserve low-voltage auxiliary power.
  • Constraints: Strict weight limits, competition ruleset compliance, and severe vibration resistance requirements.

System Architecture

Technologies Used

BMS Design ECU Interfacing MATLAB Simulation Embedded Sensors Subsystem Wiring

Key Engineering Decisions

  • Why MATLAB Simulation First? Validating the battery model in software before physical integration allowed us to size the cooling system safely without risking thermal damage to the actual cells.

Implementation

Contributed to BMS design for real-time monitoring of voltage and temperature parameters across EV battery cells. Assisted in ECU interfacing, subsystem wiring, and embedded sensor integration for the vehicle electronics. Performed MATLAB-based simulations to validate battery behaviour and model system performance under variable load conditions.

Engineering Failure Log

Failure: Battery temperature readings fluctuated wildly under physical testing.

Investigation: High-frequency mechanical vibration caused intermittent contact between the thermistors and the cell casings.

Solution: Modified the sensor mounting strategy with thermal epoxy and added software low-pass filtering to smooth the ADC readings.

Result: Stable and reliable temperature monitoring under full dynamic load conditions.

System Impact

  • Reliability: Improved confidence in battery thermal monitoring under vibration-heavy operating conditions.
  • Maintainability: Modularized the BMS wiring harness for rapid pit-stop replacement.

Prototype

Animal Deterrent System

GitHub Documentation

Overview

An ML-based real-time animal detection and deterrence system using computer vision concepts, designed to protect agricultural land from wildlife encroachment.

Problem

Crop destruction by wild animals causes significant economic loss for farmers. Traditional electric fences are dangerous, require maintenance, and are restricted in many regions.

Trade-offs & Constraints

  • Model Confidence vs. False Positives: Set a strict 85% confidence threshold to heavily penalize false positives, trading off occasional missed detections to avoid nuisance alarms.
  • Power Consumption vs. Inference Speed: Downsampled camera feed to 15fps to keep the CPU under thermal limits for passive cooling.
  • Constraints: Reliable outdoor operation under variable lighting, remote deployment with limited power budget.

System Architecture

Technologies Used

Computer Vision Machine Learning Raspberry Pi OpenCV IoT Automated Deterrent

Key Engineering Decisions

  • Why Local Edge Inference instead of Cloud? Required sub-second response times to activate the deterrent before animals caused damage, making cloud round-trip latency unacceptable. Also ensured operability in areas with zero cellular reception.

Implementation

Built an ML-based real-time animal detection pipeline using computer vision. The system processes live camera input, classifies detected animals, and triggers automated deterrent mechanisms (ultrasonic or alert-based) based on detection output. Detection logic was designed to minimise false positives in a real agricultural environment.

Engineering Failure Log

Failure: Continuous false-positive triggers during the night.

Investigation: The IR illumination caused specular reflections on wet leaves, which the baseline model misinterpreted as animal eyes/shapes.

Solution: Augmented the training dataset heavily with IR-specific agricultural background images and tuned the confidence thresholds.

Result: Nighttime false-positive rate dropped to near zero, maintaining >90% true-positive accuracy.

System Impact

  • Scalability: Proved the viability of a low-cost, edge-based alternative to expensive electric fencing.
  • Reliability: Reduced nighttime false-positive triggers to near-zero, ensuring the deterrent only fires on genuine threats.
